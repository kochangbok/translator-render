# plan.md — LLM 연동 웹페이지 번역 크롬 확장 개발 계획

## 0) 목표 정의

### 핵심 목표
- 웹페이지(영문/중문 기사)를 읽을 때 **단락(Paragraph) 단위로** 원문 바로 아래에 **한국어 번역을 삽입**하여 이중언어(원문+번역)로 표시한다.
- 번역은 **사용자 보유 LLM 구독(가능하면 OAuth 기반)** 을 최우선으로 사용하되, 불가능하면 **공식 API(키/토큰)** 로 대체한다.
- LLM이 **문서 전체 컨텍스트(제목/소제목/주변 문맥/용어 일관성)** 를 이해한 번역을 제공하도록 설계한다.
- “페이지 렌더링을 새로” 해야 한다는 요구를 만족하기 위해 **리더(Reader) 모드 기반 재렌더링**을 기본 제공(옵션으로 인라인 삽입 모드도 제공).

### 비목표(초기 버전에서 제외)
- PDF/이미지 OCR 번역
- 동영상 자막 실시간 번역
- 자동 스크롤/하이라이트/사전형 단어장(후순위)

---

## 1) 성공 기준(Definition of Done)

### 기능 성공 기준
1. **토글 동작**
   - 툴바/단축키로 번역 ON/OFF 가능
   - OFF 시 원문 레이아웃으로 복귀(리더 모드면 원문 페이지로 복귀 또는 리더 오버레이 제거)

2. **단락 단위 삽입**
   - 번역이 각 단락 바로 아래에 삽입되며, 원문 단락은 유지된다.
   - 번역 DOM 요소에는 `data-llmtr` 같은 명확한 attribute가 붙고, 원문과 1:1 매핑 가능.

3. **컨텍스트 유지 번역**
   - 동일 인명/기관/용어가 문서 내에서 일관되게 번역된다(글로서리/번역 메모리).
   - 문단 단위 처리라도 최소한 **이웃 문단 문맥**이 반영된다.

4. **재렌더링(Reader Mode)**
   - 기사 본문을 추출해(Reader/Readability) 새로운 컨테이너(오버레이/탭)로 재렌더링하고,
     그 안에서 이중언어를 구현한다.
   - 원페이지에 직접 삽입(인라인 모드)은 선택 옵션.

5. **안정성**
   - SPA/무한스크롤 페이지에서 DOM이 변해도 번역 레이어가 크게 깨지지 않는다(관찰+재주입).
   - 번역 요청이 실패하면 단락별로 재시도/에러 표시가 된다.

### 품질/성능 성공 기준(권장)
- 2,000~5,000자 기사 기준 “번역 시작 → 상단 5개 문단 번역 표시”까지 3~8초 내(네트워크/모델에 따라 가변).
- 동일 URL 재방문 시 캐시로 70% 이상 즉시 표시(해시 기반).

## 2) 주요 의사결정 포인트(초기 가정)
- **가정 A:** “구독 요금제 OAuth”가 공식적으로 제공되지 않는 LLM(예: 일부 소비자용 챗 UI 구독)인 경우가 많다.  
  ⇒ **공식 API 키(또는 자체 백엔드 프록시) 경로를 기본 플랜으로 포함**한다.
- **가정 B:** 리더 모드 재렌더링이 요구사항의 “페이지 렌더링 새로”를 가장 확실히 만족.  
  ⇒ **MVP 기본 UX = Reader Mode**, 인라인 삽입은 M1 이후 옵션.
- **가정 C:** 모델/비용/속도는 사용자가 옵션 페이지에서 선택하도록 설계.

## 3) 단계별 개발 로드맵(Agentic Vibe Coding 친화)

### M0 — 스캐폴딩 & 최소 동작(1차 뼈대)
- [ ] MV3 확장 스캐폴딩(Typescript 권장)
  - [ ] `manifest.json` (permissions/host_permissions, action, options, background service worker)
  - [ ] `src/background/` 서비스워커
  - [ ] `src/content/` 콘텐츠 스크립트
  - [ ] `src/ui/popup/`, `src/ui/options/`
- [ ] 페이지에서 “기사 본문 텍스트”를 추출
  - [ ] 우선 Readability 기반(실패 시 간단 fallback)
- [ ] 문단 목록(Paragraph blocks) 생성(각 블록에 stable id 부여)
- [ ] 더미 번역(예: "[KO] ..." prepend) 삽입
- [ ] ON/OFF 토글 + 원복

**M0 합격 테스트**
- 임의 블로그/뉴스 페이지 3개에서 리더 모드 오버레이가 뜨고 문단별 더미 번역이 원문 아래 출력.

---

### M1 — LLM 번역 파이프라인(컨텍스트/구조화 출력)
- [ ] Provider 추상화 인터페이스 정의(`ProviderAdapter`)
- [ ] 1개 Provider 구현(우선순위: 사용자가 실제 쓸 수 있는 것)
  - [ ] API Key 방식(기본)
  - [ ] OAuth 가능 Provider는 이후(또는 병렬)
- [ ] 문서 컨텍스트 구성
  - [ ] title, headings, url, domain, 추출된 본문 텍스트
  - [ ] 글로서리 후보(고유명사/용어) 추출(간단 규칙 또는 LLM 1회)
- [ ] 번역 요청 설계
  - [ ] **chunk 단위**(N문단 묶음)로 번역하여 컨텍스트 확보
  - [ ] 구조화 JSON 반환(문단 id 매핑)
- [ ] 스트리밍/점진 표시(가능하면)
- [ ] 실패/재시도/속도 제한(큐)

**M1 합격 테스트**
- 영문 기사 2개, 중문 기사 2개에서 문단별 한국어 번역이 자연스럽게 삽입되고, 문서 내 용어가 대체로 일관.

---

### M2 — OAuth(가능한 경우) + 키/프록시 하이브리드
- [ ] “구독 요금제 OAuth” 가능성 조사 결과를 반영해 구현 경로 확정
  - [ ] (가능) `chrome.identity` 또는 `launchWebAuthFlow` 기반 OAuth
  - [ ] (불가/제약) API Key 입력 + (선택) 사용자 개인 백엔드 프록시
- [ ] 토큰 저장/갱신 전략(secure storage)
- [ ] Provider별 레이트리밋/에러 매핑

**M2 합격 테스트**
- 사용자가 옵션에서 로그인/키설정을 완료하면 이후 페이지에서 별도 입력 없이 번역이 동작.

---

### M3 — 인라인 삽입 모드(원 페이지 DOM에 삽입) + 내구성
- [ ] 인라인 모드: 원문 DOM에서 `p, li` 등을 직접 매핑하고 번역 삽입
- [ ] SPA/동적 로딩 대응
  - [ ] MutationObserver + URL change 감지
  - [ ] 중복 삽입 방지(마커)
- [ ] 사용자 선택 기반 번역 범위
  - [ ] (옵션) “본문만 / 전체 페이지 / 선택 영역” 모드

**M3 합격 테스트**
- Medium/Notion 스타일 SPA 페이지 2개에서 스크롤 로딩 중에도 번역 삽입이 유지.

---

### M4 — 품질/비용 최적화(캐시/용어집/평가)

## 5) 채점용 스코어카드(구현 결과물 평가 기준)

> 총점 100점. 아래 항목을 “실제로 동작하는지” 기준으로 채점 가능하게 설계.

### A. 핵심 UX (25)
- (10) 번역 ON/OFF 토글 + 원복 완전 동작
- (10) 문단 아래 삽입 1:1 매핑(중복/누락 없음)
- (5) 스타일(가독성, 접기/펼치기, 로딩 표시)

### B. 재렌더링(Reader Mode) (15)
- (10) Readability 기반 기사 추출 & 깨짐 최소
- (5) 원페이지 복귀/오버레이 제거 안정

### C. LLM 컨텍스트 품질 (25)
- (10) 제목/헤딩 포함 컨텍스트 전달
- (10) 용어 일관성(글로서리/메모리) 체감
- (5) 구조화 출력(JSON)로 문단 매핑 안정

### D. Provider/Auth (15)
- (10) 최소 1개 Provider 완전 동작(API 키 또는 OAuth)
- (5) 오류 메시지/재시도/토큰 만료 처리

### E. 성능/캐시/내구성 (20)
- (10) 캐시(재방문/새로고침 시 재사용)
- (5) 동적 DOM 대응(MutationObserver 등)
- (5) 레이트리밋/큐(과도 요청 방지)

## 6) 리스크 & 완화
- **OAuth로 “구독 요금제” 직접 사용**은 공식 지원이 없는 경우가 흔함 → API Key/백엔드 프록시를 항상 플랜B로 유지.
- 일부 사이트의 복잡한 DOM/SPA로 본문 추출 실패 가능 → 인라인 모드 + 사용자 선택(“이 영역을 본문으로”) 제공.
- 비용 폭증 위험 → 기본 토큰 제한, 모델 기본값(저비용), 캐시, 문단 길이 컷오프.
